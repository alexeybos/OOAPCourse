В Java поддерживаются следующие принципы:
1. Дженерики - поддерживаются.
2. Объединение несколько функций, которые активно обращаются друг к другу.
   Если я не ошибаюсь в трактовке этого принципа - это package-private в пакетах. Принцип поддерживается.
3. Семейство модулей, ориентированных на решение некоторой общей задачи.
   Это иерархичная структура пакетов - поддерживается.
4. Динамический выбор реализации.
   Насколько я понимаю это поддерживается уже на уровне классов, а не на уровне "базовой синтаксической единицы"
5. Интеграция общего поведения модулей
   Java позволяет создавать абстрактные классы и интерфейсы, которые могут быть реализованы различными классами. Но это тоже на уровне классов.
